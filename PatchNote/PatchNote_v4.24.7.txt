v4.24.1 -> v4.24.7

パッチアップデート:

1. キャリブレーション計算の安全性を改善
マーカーサイズに 0、負の値、NaN、inf が含まれている場合、log10 計算へ進まないようにしました。
不正なマーカー値がある場合は検量線係数とサンプルサイズを未計算状態に戻し、不正な検量線が作成されないようにしました。
Excel / CSV 出力時の log10 計算にも同じチェックを追加しました。

2. ドラッグ操作の軽量化
開始ライン、終了ライン、マーカー、サンプルのドラッグ中は、重い検量線再計算・結果テーブル更新・レイヤーパネル再構築を行わず、マウスを離した時にまとめて更新するようにしました。
ドラッグ中のキャンバス描画は予約式にして、マウスイベントが大量に発生しても描画処理が積み上がらないようにしました。
レイヤーパネルのドラッグ選択では、行一覧をキャッシュし、同じ行上での不要な選択更新を省略するようにしました。

3. 画像表示と読み込みのメモリ負荷を改善
画像読み込み時にファイル全体を一度バイト配列へ読み込む処理をやめ、Pillow にファイルパスを直接渡す方式に変更しました。
キャンバス表示用のリサイズ画像をキャッシュし、パン操作時の不要な再リサイズを減らしました。
Undo 履歴では、画像そのものが変更される回転・トリミングなどの場合だけ画像コピーを保持し、注釈移動などでは大きな画像コピーを避けるようにしました。

4. サイレントエラーの削減
複数箇所の `except Exception: pass` を見直し、ログにエラー内容が残るようにしました。
これにより、UI 更新やドラッグ&ドロップ初期化などで問題が起きた場合に原因を追跡しやすくなりました。

5. フォント依存の改善
UI 上でハードコードされていた Helvetica 指定を Tk の標準フォント指定へ置き換えました。
環境によって Helvetica が存在しない場合でも、表示が崩れにくくなりました。

6. マーカープリセットの堅牢性を改善
プリセット保存時に 0 以下のサイズを除外し、ログに記録するようにしました。
手動編集されたプリセットファイルに不正なサイズが混じっていても、キャリブレーション計算へ悪影響が出にくくなりました。


Patch Updates:

1. Improved calibration calculation safety
Calibration no longer proceeds to log10 calculation when marker sizes contain 0, negative values, NaN, or inf.
When invalid marker values are detected, calibration coefficients and sample sizes are reset to an uncalculated state so invalid calibration curves are not created.
The same validation was also added to log10 calculations during Excel / CSV export.

2. Improved drag performance
While dragging the start line, end line, markers, or samples, heavy calibration recalculation, result table updates, and layer panel rebuilds are skipped and applied once when the mouse button is released.
Canvas redraw during dragging is now scheduled, preventing redraw operations from piling up when many mouse events are generated.
Layer panel drag selection now caches the row list and skips unnecessary selection updates while the pointer remains on the same row.

3. Reduced image display and loading memory overhead
Image loading no longer reads the entire file into a byte array first; Pillow now opens the image directly from the file path.
Resized canvas preview images are cached to reduce unnecessary resizing during pan operations.
Undo history now stores image copies only when the image itself changes, such as rotation or trimming, while annotation-only operations avoid large image copies.

4. Reduced silent error handling
Several `except Exception: pass` blocks were replaced with logging.
This makes issues during UI updates, drag-and-drop initialization, and related operations easier to trace.

5. Improved font portability
Hardcoded Helvetica font usage in the UI was replaced with Tk's default font setting.
This reduces layout issues on systems where Helvetica is unavailable.

6. Improved marker preset robustness
Marker presets now ignore sizes that are 0 or below when saving, and log those invalid values.
This helps prevent manually edited preset files with invalid sizes from affecting calibration calculations.
