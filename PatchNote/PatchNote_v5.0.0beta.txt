パッチアップデート:

1. キャリブレーション計算の安全性を改善
マーカーサイズに 0、負の値、NaN、inf が含まれている場合、log10 計算へ進まないようにしました。
不正なマーカー値がある場合は検量線係数とサンプルサイズを未計算状態に戻し、不正な検量線が作成されないようにしました。
Excel / CSV 出力時の log10 計算にも同じチェックを追加しました。

2. ドラッグ操作の軽量化
開始ライン、終了ライン、マーカー、サンプルのドラッグ中は、重い検量線再計算・結果テーブル更新・レイヤーパネル再構築を行わず、マウスを離した時にまとめて更新するようにしました。
ドラッグ中のキャンバス描画は予約式にして、マウスイベントが大量に発生しても描画処理が積み上がらないようにしました。
レイヤーパネルのドラッグ選択では、行一覧をキャッシュし、同じ行上での不要な選択更新を省略するようにしました。

3. 画像表示と読み込みのメモリ負荷を改善
画像読み込み時にファイル全体を一度バイト配列へ読み込む処理をやめ、Pillow にファイルパスを直接渡す方式に変更しました。
キャンバス表示用のリサイズ画像をキャッシュし、パン操作時の不要な再リサイズを減らしました。
Undo 履歴では、画像そのものが変更される回転・トリミングなどの場合だけ画像コピーを保持し、注釈移動などでは大きな画像コピーを避けるようにしました。

4. サイレントエラーの削減
複数箇所の `except Exception: pass` を見直し、ログにエラー内容が残るようにしました。
これにより、UI 更新やドラッグ&ドロップ初期化などで問題が起きた場合に原因を追跡しやすくなりました。

5. フォント依存の改善
UI 上でハードコードされていた Helvetica 指定を Tk の標準フォント指定へ置き換えました。
環境によって Helvetica が存在しない場合でも、表示が崩れにくくなりました。

6. マーカープリセットの堅牢性を改善
プリセット保存時に 0 以下のサイズを除外し、ログに記録するようにしました。
手動編集されたプリセットファイルに不正なサイズが混じっていても、キャリブレーション計算へ悪影響が出にくくなりました。


v4.24.1 -> v5.0.0beta

Major Update:
1. Automatic Band Detection (Unified Framework for Markers and Samples)
A unified framework supporting both markers and samples has been implemented as "Auto-Add Mode." When a user selects a lane by dragging, the system automatically detects bands using an algorithm robust against "smiling" distortion and confirms the results via an approval UI.

(A). Handling "Smiling" Gel Distortion (Center-Strip Scanning)
Since vertical scanning across the full lane width can lead to a single band being misidentified as multiple peaks, vertical profiles are generated—and peak detection performed—using only the central strip of the lane.

(B). Handling Thick Bands and Flat-Topped Peaks (Centroid Detection)
By determining the registered coordinates based on the intensity centroid, the system converges on a single point even for trapezoidal or rectangular peak profiles. (C). Approval UI
・Detection candidates: Displayed with orange dotted-line rectangles (crosshair marker at the centroid coordinates).
・Approved: Changes to a green solid-line frame.
・Rejected: Changes to a red frame (can be restored later).
・Batch operations via "Approve All" and "Reject All" buttons.

*Detailed specifications subject to revision.


Minor Updates:
1. PDF Output
・Output layout: p.1 = Gel image with annotations; p.2 = Calibration curve graph; p.3 = Marker & sample results table.
・Paper size selection: A4 (default) / Letter.
・Japanese text support (shares matplotlib Japanese font settings with `fonts.py`).
・Implemented using matplotlib's PDF backend instead of reportlab (zero additional dependencies).

2. Densitometry (Band Intensity Quantification)
・Mode allowing rectangular ROI selection by clicking and dragging after selecting a band.
・Real-time display of the vertical intensity profile within the ROI in the side panel.
・Background correction: Subtracts rolling background based on the average intensity at the top and bottom edges of the ROI.
・Displays Integrated Density and normalized relative values ​​(ratio to the maximum band) in the panel.
・Added a "Densitometry Sheet" to Excel export (includes ROI coordinates, integrated values, and relative values).

3. Batch Comparison View for Multiple Lanes
A feature to overlay and graph intensity profiles from multiple lanes. Facilitates visual comparison of band shifts and intensity differences.
・Overlays intensity profiles for each lane using Matplotlib (lane/sample names in the legend).
・Lanes to display can be selected via checkboxes.
・Graphs can be exported as PNG or SVG.
・Adds vertical lines to the intensity graph at positions corresponding to markers and sample Rf values.
・Moving the vertical lines synchronizes with the main window.

4. Lane Comparison Mode
A view that displays multiple lanes from the same gel side-by-side for comparison. Organizes and labels which sample is located in which lane.・Open a dedicated window via the menu: "View → Lane Comparison Mode."
・Display lanes side-by-side as vertically oriented thumbnails.
・Show sample name labels above each lane (editable).
・Align band positions across lanes using horizontal guidelines (optional).
・Includes PNG export functionality (format suitable for pasting comparison figures into papers or reports).

Patch Update:

1. Improved safety of calibration calculations
Prevented the system from proceeding to log10 calculations if marker sizes include 0, negative values, NaN, or inf.
If invalid marker values ​​are detected, calibration curve coefficients and sample sizes are reset to an uncalculated state to prevent the creation of invalid calibration curves.
Added the same checks for log10 calculations during Excel/CSV output.

2. Optimized drag operations
During the dragging of start lines, end lines, markers, or samples, resource-intensive tasks—such as recalculating calibration curves, updating result tables, and rebuilding the layer panel—are deferred until the mouse button is released.
Canvas rendering during dragging now uses a queued approach to prevent rendering processes from piling up when a large number of mouse events occur.
For drag-selection in the layer panel, the row list is now cached to skip redundant selection updates on the same row.

3. Reduced memory load for image display and loading
Changed the image loading process to pass the file path directly to Pillow, eliminating the step of reading the entire file into a byte array first.
Cached resized images for canvas display to reduce unnecessary resizing during panning operations.
In the undo history, image copies are now retained only for operations that alter the image itself (e.g., rotation, cropping), while avoiding large image copies for operations like moving annotations.

4. Reduced silent errors
Reviewed multiple instances of `except Exception: pass` and updated them to ensure error details are recorded in the logs.
This makes it easier to trace the cause of issues occurring during UI updates, drag-and-drop initialization, and other processes. 5. Improved font dependency handling
Replaced the hard-coded "Helvetica" font specification in the UI with Tk's standard font specification.
This reduces the likelihood of display issues in environments where Helvetica is unavailable.

6. Improved robustness of marker presets
Modified the preset saving process to exclude sizes of 0 or less and log them.
This prevents invalid sizes in manually edited preset files from adversely affecting calibration calculations.
